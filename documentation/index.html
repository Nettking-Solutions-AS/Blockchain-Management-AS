A) System Requirement - top

  • Operating system : Linux
  • Docker
  • Web3 cli
  • Nodejs : 12.x +
  • MongoDB : 4.x +
  • Angular : 11.x +
  • SSH root access for installation purpose

B) How to Web3 CLI - top

  • Refer offical documentation for docker installation in linux. click here
  • Install web3 cli using below command from terminal 

    curl -LSs https://raw.githubusercontent.com/gochain/web3/master/install.sh | sh

  • Create or login infura.io account
  • Go to ethereum dashboard. please refer below link 

    https://infura.io/dashboard/ethereum

  • Create new project and go to settings. please see screenshot
  • Copy http endpoint from infura. screenshot
  • update WEB3_RPC_URL in following files from package root directory
    1. create.sh file in ./backend directory
    2. send.sh file in ./backend directory
    3. transaction.sh file in ./backend directory
    4. mint.sh file in ./backend directory
    5. address.sh file in ./backend directory

C) How to configure mongoDB - top

  • Install mongodb in linux with offical documentation. please refer by clicking the link
  • Go to ./backend/helper/config.js from application root directory
  • Update database information. see the link
    1. Host - it will be mongodb running host. By default it was running in localhost
    2. Port - it will be mongodb running port. By default it was running in 27017
    3. Name - it will be database name
    4. Prefix - it will be database collection prefix. it was option variable

D) How to configure mail - top

  • SMTP configuration
    1. Create SMTP details in any mail provider
    2. Go to ./backend/helper/config.js from application root directory
    3. update SMTP detail under mail variable. see screenshot here
    4. update type to "smtp". see screenshot here
  • Follow below steps to avoid email notification in the application
    1. Go to ./backend/helper/config.js from application root directory
    2. update type to "" (empty string). see screenshot here

E) Permissions - top

  • Go to ./backend from application root directory
  • Give full permission for media folder in linux 

    sudo chmod 777 -R media

F) Install and run nodejs backend - top

  • Go to ./backend from application root directory in terminal
  • Run "npm install" from terminal
  • To run the application temporarily. Run following command 

    npm run dev

  • To run the the application always. Follow below steps
    1. Install pm2 by running the following command 

      sudo npm install -g pm2

    2. Go to ./backend from application root directory in terminal 

      pm2 start 'npm run dev' --name backend

    3. Run "example.com/settings/install" to configure admin user and admin comission settings. make sure change "example.com" with valid api host information
    4. Go to ./backend/module/comment/route/settings.js from application root directory. and comment or remove following line 

      router.get('/install',settingsController.installOptions)

G) Install and run angular frontend - top

  • Go to ./frontend from application root directory in terminal
  • Run "npm install" from terminal
  • Go to ./frontend/src/app/constants/config.ts and ./frontend/src/app/constants/api.ts from application root directory
  • replace "example.com" with correct backend url. please see screenshot
  • To run the the application always. Follow below steps
    1. Install pm2 by running the following command "sudo npm install -g pm2"
    2. Go to ./frontend from application root directory in terminal
    3. Run "pm2 start 'npm run dev' --name backend"
  • Go to ./frontend from application root directory in terminal
  • To run the application use below command pm2 start "ng serve --host 0.0.0.0 --port 5001 --disable-host-check --prod --live-reload false --name frontend
  • Make sure to enable the ports used in both frontend and backend in hosting firewall or security groups

I) How to configure MultiSend Contract Address for mainnet - top

  • Go to .backend/contracts and remove MutlSend.bin and MultiSend.abi files
  • Update WEB3_PRIVATE_KEY with mainnet private key, make sure you have enough gas fee to compile MultiSend.sol file "export WEB3_PRIVATE_KEY=a8f617ed9d45404af91bdacaf901fb9bb60669f6c57bc969597301277725faf3"
  • Run MutliSend.sh file from terminal and it wil give new contract address
  • Copy the MultiSend contract address and update in frontend folder "src/app/constants/config.ts" file. Please see screenshot

J) How to configure Admin address to receive commission and transfer asset after sale - top

  • Open "src/app/constants/config.ts" from frotend and update "main_address" with mainnet admin address. Please see screenshot

K) How to configure mainnet for frontend - top

  • Open "src/app/constants/config.ts" from frotend and update "blockchain_url" to https://etherscan.io/
  • Open "src/app/constants/config.ts" from frotend and update "blockchain" to "Mainnet"
  • Open "src/app/constants/config.ts" from frotend and update "onboard" network to 1, Please see screenshot
  • Create new blocknative account and update "onboard" key - https://prnt.sc/1wdngtf. To generate new onboard key please follow this tutorial https://docs.blocknative.com/onboard

L) How to configure mainnet for backend - top

  • Goto backend and open address.sh file and replace following string with mainnet rpc url. you can do the same setup with create.sh, generate.sh, mint.sh, send.sh, transaction.sh "export WEB3_RPC_URL=https://rinkeby.infura.io/v3/64fa77a39b9a4c31b186fb2148edff70"
  • Open helper/config.js and update eth_http with mainnet rpc url - Please see screenshot
  • Open helper/config.js and update owner key with mainnet private key and public key - https://prnt.sc/1wdoewt. use same mainnet address in both backend and frontend
